## 為什麼先教這個？

不寫測試的程式碼是「可能會動」，有測試的程式碼是「確定會動」。測試是修改程式碼後不會造成回歸錯誤的安全網——這是從「學習專案」走向「生產專案」的關鍵一步。

## 核心觀念

### 測試金字塔

```
          ╱ ╲
         ╱   ╲          e2e 測試
        ╱ 少  ╲         啟動完整應用 + 真實 DB
       ╱ 且慢  ╲        速度：慢（秒級）
      ╱─────────╲
     ╱           ╲       整合測試
    ╱   中等量    ╲      部分模組 + Mock
   ╱    中等速    ╲      速度：中（百毫秒級）
  ╱───────────────╲
 ╱                 ╲     單元測試
╱    多且快         ╲    純 TypeScript，不啟動應用
╱───────────────────╲    速度：快（毫秒級）
```

> **策略**：先追求底層（單元測試）覆蓋率，再補上 e2e 測試驗證完整流程。

### Jest 核心概念

| API | 用途 | 範例 |
|-----|------|------|
| `jest.fn()` | 建立 mock function | `const mockFn = jest.fn()` |
| `jest.spyOn()` | 監視既有方法 | `jest.spyOn(service, 'findById')` |
| `mockResolvedValue()` | 定義 async mock 的回傳值 | `.mockResolvedValue(book)` |
| `mockReturnValue()` | 定義 sync mock 的回傳值 | `.mockReturnValue('value')` |
| `mockRejectedValue()` | 定義 async mock 拋出的錯誤 | `.mockRejectedValue(new Error())` |
| `expect().toHaveBeenCalledWith()` | 驗證呼叫參數 | `expect(fn).toHaveBeenCalledWith(1)` |

### @nestjs/testing

NestJS 提供的測試模組，可建立隔離的測試環境：

```typescript
import { Test, TestingModule } from '@nestjs/testing';

const module: TestingModule = await Test.createTestingModule({
  providers: [
    BookService,
    {
      provide: BookRepository,
      useValue: mockBookRepository,  // 用 Mock 替換真實 Repository
    },
  ],
}).compile();

const service = module.get<BookService>(BookService);
```

### AAA 模式

每個測試方法都遵循 **Arrange → Act → Assert** 三步驟：

```typescript
it('should return book when found', async () => {
  // Arrange：準備測試資料與 Mock 行為
  const mockBook = { id: 1, title: 'Test' };
  jest.spyOn(repository, 'findById').mockResolvedValue(mockBook);

  // Act：執行被測方法
  const result = await service.findById(1);

  // Assert：驗證結果
  expect(result.id).toBe(1);
  expect(repository.findById).toHaveBeenCalledWith(1);
});
```

### `book.service.spec.ts` 完整範例

```typescript
import { Test, TestingModule } from '@nestjs/testing';
import { NotFoundException } from '@nestjs/common';
import { BookService } from './book.service';
import { BookRepository } from './book.repository';
import { CacheService } from '../common/cache/cache.service';
import { Book } from './entities/book.entity';

describe('BookService', () => {
  let service: BookService;
  let repository: jest.Mocked<BookRepository>;
  let cacheService: jest.Mocked<CacheService>;

  beforeEach(async () => {
    // 建立 Mock 物件
    const mockRepository = {
      findAll: jest.fn(),
      findById: jest.fn(),
      create: jest.fn(),
      update: jest.fn(),
      delete: jest.fn(),
    };

    const mockCacheService = {
      get: jest.fn(),
      set: jest.fn(),
      del: jest.fn(),
    };

    const module: TestingModule = await Test.createTestingModule({
      providers: [
        BookService,
        { provide: BookRepository, useValue: mockRepository },
        { provide: CacheService, useValue: mockCacheService },
      ],
    }).compile();

    service = module.get<BookService>(BookService);
    repository = module.get(BookRepository);
    cacheService = module.get(CacheService);
  });

  describe('findById', () => {
    const sampleBook: Book = {
      id: 1,
      title: 'Test Book',
      author: 'Author',
      isbn: '1234567890',
      stock: 5,
      createdAt: new Date(),
      updatedAt: new Date(),
    };

    it('should return book from cache when cache hit', async () => {
      // Arrange：快取命中
      cacheService.get.mockResolvedValue(sampleBook);

      // Act
      const result = await service.findById(1);

      // Assert：應該從快取取得，不查 DB
      expect(result.id).toBe(1);
      expect(cacheService.get).toHaveBeenCalledWith('book:1');
      expect(repository.findById).not.toHaveBeenCalled();
    });

    it('should query DB and cache result when cache miss', async () => {
      // Arrange：快取未命中，DB 有資料
      cacheService.get.mockResolvedValue(null);
      repository.findById.mockResolvedValue(sampleBook);

      // Act
      const result = await service.findById(1);

      // Assert：查 DB 並寫入快取
      expect(result.id).toBe(1);
      expect(repository.findById).toHaveBeenCalledWith(1);
      expect(cacheService.set).toHaveBeenCalled();
    });

    it('should throw NotFoundException when book not found', async () => {
      // Arrange：快取未命中，DB 也查無資料
      cacheService.get.mockResolvedValue(null);
      repository.findById.mockResolvedValue(null);

      // Act + Assert：預期拋出 NotFoundException
      await expect(service.findById(99)).rejects.toThrow(NotFoundException);
    });
  });

  describe('update', () => {
    it('should update book and evict cache', async () => {
      // Arrange
      const updatedBook: Book = { ...sampleBook, title: 'Updated' };
      repository.findById.mockResolvedValue(sampleBook);
      repository.update.mockResolvedValue(updatedBook);

      // Act
      const dto = { title: 'Updated', author: 'Author', isbn: '123', stock: 5 };
      await service.update(1, dto);

      // Assert：更新後應清除快取
      expect(repository.update).toHaveBeenCalledWith(1, dto);
      expect(cacheService.del).toHaveBeenCalledWith('book:1');
    });
  });

  describe('delete', () => {
    it('should delete book and evict cache', async () => {
      // Act
      await service.delete(1);

      // Assert
      expect(repository.delete).toHaveBeenCalledWith(1);
      expect(cacheService.del).toHaveBeenCalledWith('book:1');
    });
  });
});
```

### `book.api.controller.spec.ts` Controller 測試

```typescript
import { Test, TestingModule } from '@nestjs/testing';
import { BookApiController } from './book.api.controller';
import { BookService } from './book.service';
import { NotFoundException } from '@nestjs/common';

describe('BookApiController', () => {
  let controller: BookApiController;
  let service: jest.Mocked<BookService>;

  beforeEach(async () => {
    const mockService = {
      findAll: jest.fn(),
      findById: jest.fn(),
      create: jest.fn(),
      update: jest.fn(),
      delete: jest.fn(),
    };

    const module: TestingModule = await Test.createTestingModule({
      controllers: [BookApiController],
      providers: [{ provide: BookService, useValue: mockService }],
    }).compile();

    controller = module.get<BookApiController>(BookApiController);
    service = module.get(BookService);
  });

  describe('findById', () => {
    it('should return book when found', async () => {
      const mockBook = { id: 1, title: 'Test' };
      service.findById.mockResolvedValue(mockBook as any);

      const result = await controller.findById('1');

      expect(result.id).toBe(1);
      expect(service.findById).toHaveBeenCalledWith(1);
    });

    it('should throw NotFoundException when not found', async () => {
      service.findById.mockRejectedValue(new NotFoundException());

      await expect(controller.findById('99')).rejects.toThrow(NotFoundException);
    });
  });
});
```

### e2e 測試 — `test/app.e2e-spec.ts`

```typescript
import { Test, TestingModule } from '@nestjs/testing';
import { INestApplication, ValidationPipe } from '@nestjs/common';
import * as request from 'supertest';
import { AppModule } from '../src/app.module';

describe('BookApiController (e2e)', () => {
  let app: INestApplication;

  beforeAll(async () => {
    const moduleFixture: TestingModule = await Test.createTestingModule({
      imports: [AppModule],
    }).compile();

    app = moduleFixture.createNestApplication();
    app.useGlobalPipes(new ValidationPipe());
    await app.init();
  });

  afterAll(async () => {
    await app.close();
  });

  describe('GET /api/books', () => {
    it('should return 200 and array of books', () => {
      return request(app.getHttpServer())
        .get('/api/books')
        .expect(200)
        .expect((res) => {
          expect(Array.isArray(res.body)).toBe(true);
        });
    });
  });

  describe('POST /api/books', () => {
    it('should return 201 when valid data', () => {
      return request(app.getHttpServer())
        .post('/api/books')
        .send({
          title: 'New Book',
          author: 'Author',
          isbn: '9999999999',
          stock: 10,
        })
        .expect(201)
        .expect((res) => {
          expect(res.body.title).toBe('New Book');
        });
    });

    it('should return 400 when validation fails', () => {
      return request(app.getHttpServer())
        .post('/api/books')
        .send({ author: 'Author' })  // 缺少 title
        .expect(400);
    });
  });

  describe('GET /api/books/:id', () => {
    it('should return 404 when not found', () => {
      return request(app.getHttpServer())
        .get('/api/books/99999')
        .expect(404);
    });
  });
});
```

### 單元測試 vs e2e 測試

| 特性 | 單元測試 | e2e 測試 |
|------|----------|----------|
| 速度 | 快（毫秒） | 慢（秒） |
| 依賴 | Mock 所有依賴 | 真實 DB / Redis |
| 範圍 | 單一類別/方法 | 完整請求流程 |
| 用途 | 驗證邏輯正確 | 驗證整合正確 |
| 執行指令 | `npm run test` | `npm run test:e2e` |

## 踩坑提示

| 現象 | 原因 | 解法 |
|------|------|------|
| Mock 物件全是 undefined，測試直接報錯 | 忘記設定 mock 的回傳值 | 確保所有被呼叫的 mock 方法都有設定 `mockResolvedValue()` 或 `mockReturnValue()` |
| `mockResolvedValue` 設定了但還是回傳 undefined | Mock 的方法名稱打錯或參數不匹配 | 確認方法名稱完全正確；必要時用 `console.log` debug |
| e2e 測試連不上資料庫 | 測試環境沒有啟動 DB 容器 | e2e 測試前先 `docker compose up -d` |
| e2e 測試互相干擾 | 測試資料沒有隔離 | 每個測試後清理資料，或使用 transaction rollback |
