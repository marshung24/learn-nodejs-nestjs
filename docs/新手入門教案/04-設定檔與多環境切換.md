## ② 對應檔案

| 檔案 | 角色 |
|------|------|
| `.env` / `.env.example` | 環境變數檔 |
| `src/config/database.config.ts` | TypeORM 連線設定 |
| `src/config/redis.config.ts` | Redis 連線設定 |
| `src/app.module.ts` | ConfigModule 載入設定 |
| `docker-compose.yml` | 多容器編排 |

---

## ③ 核心觀念

### `.env` 檔與環境變數

- 用 `@nestjs/config` 的 `ConfigModule` 載入
- 敏感資訊（密碼、API Key）不應 commit 到 Git

### ConfigService

- 依賴注入取得設定值
- 避免直接讀取 `process.env`

```typescript
constructor(private configService: ConfigService) {}

getPort(): number {
  return this.configService.get<number>('PORT', 3000);
}
```

### 設定優先順序

1. 環境變數（最高）
2. `.env` 檔
3. 預設值（最低）

### Docker Compose

- 一個 `docker compose up -d` 拉起 App + MySQL + Redis
- 三個容器互相連接

---

## ④ 動手做

### 必做

1. **啟動完整環境**
   - 執行 `docker compose up -d`
   - 用 `docker compose ps` 確認三個容器都在 running 狀態

2. **閱讀設定檔**
   - 閱讀 `database.config.ts`
   - 說明如何從 `ConfigService` 取得設定值

### 延伸挑戰

1. 新增一個自訂環境變數 `APP_NAME`
   - 在 Controller 中用 `ConfigService` 讀取並回傳

## ⑤ 踩坑提示

| 現象 | 原因 | 解法 |
|------|------|------|
| `ConfigService.get()` 回傳 undefined | `.env` 檔放錯位置或變數名稱打錯 | 確認 `.env` 在專案根目錄 |
| `docker compose up` 後 App 啟動失敗 | MySQL 容器尚未 ready | 等 healthcheck 通過後重試 |
