## 核心觀念

### `.env` 檔與環境變數

使用 `@nestjs/config` 的 `ConfigModule` 載入環境變數：

**`.env.example`**（範本檔，應 commit 到 Git）：

```env
# 應用程式
PORT=3000

# 資料庫
DATABASE_HOST=localhost
DATABASE_PORT=3306
DATABASE_USER=app
DATABASE_PASSWORD=app
DATABASE_NAME=app

# Redis
REDIS_HOST=localhost
REDIS_PORT=6379
```

**`.env`**（實際設定檔，不應 commit 到 Git）：

從 `.env.example` 複製並修改：

```bash
cp .env.example .env
```

### ConfigModule 設定

在 `AppModule` 中載入 ConfigModule：

```typescript
import { Module } from '@nestjs/common';
import { ConfigModule } from '@nestjs/config';

@Module({
  imports: [
    ConfigModule.forRoot({
      isGlobal: true,  // 全域可用，不需要在每個 Module 重複 import
      envFilePath: '.env',
    }),
    // ... 其他模組
  ],
})
export class AppModule {}
```

### ConfigService — 依賴注入取得設定值

避免直接讀取 `process.env`，改用 `ConfigService`：

```typescript
import { Injectable } from '@nestjs/common';
import { ConfigService } from '@nestjs/config';

@Injectable()
export class AppService {
  constructor(private configService: ConfigService) {}

  getPort(): number {
    // 第二個參數是預設值
    return this.configService.get<number>('PORT', 3000);
  }

  getDatabaseHost(): string {
    return this.configService.get<string>('DATABASE_HOST', 'localhost');
  }
}
```

### 設定優先順序

環境變數 > `.env` 檔 > 預設值

```
1. 系統環境變數（最高）  → export PORT=4000
2. .env 檔              → PORT=3000
3. ConfigService 預設值  → configService.get('PORT', 3000)
```

### database.config.ts — TypeORM 連線設定

```typescript
import { Injectable } from '@nestjs/common';
import { ConfigService } from '@nestjs/config';
import { TypeOrmModuleOptions, TypeOrmOptionsFactory } from '@nestjs/typeorm';

@Injectable()
export class DatabaseConfigService implements TypeOrmOptionsFactory {
  constructor(private configService: ConfigService) {}

  createTypeOrmOptions(): TypeOrmModuleOptions {
    return {
      type: 'mysql',
      host: this.configService.get<string>('DATABASE_HOST', 'localhost'),
      port: this.configService.get<number>('DATABASE_PORT', 3306),
      username: this.configService.get<string>('DATABASE_USER', 'app'),
      password: this.configService.get<string>('DATABASE_PASSWORD', 'app'),
      database: this.configService.get<string>('DATABASE_NAME', 'app'),
      entities: [__dirname + '/../**/*.entity{.ts,.js}'],
      synchronize: false,  // 生產環境絕對不要開！用 migration 管理 schema
    };
  }
}
```

### Docker Compose 一鍵啟動

`docker-compose.yml` 定義三個服務：

```yaml
services:
  app:
    build:
      context: .
      dockerfile: docker/node/Dockerfile
    depends_on:
      db:    { condition: service_healthy }
      redis: { condition: service_healthy }
    environment:
      DATABASE_HOST: db          # 容器內使用服務名稱連接
      REDIS_HOST: redis
    ports:
      - "3000:3000"
    volumes:
      - .:/app                    # 開發時 bind mount，支援 hot reload
      - /app/node_modules         # 排除 node_modules

  db:
    image: mysql:8.0
    environment:
      MYSQL_ROOT_PASSWORD: root
      MYSQL_DATABASE: app
      MYSQL_USER: app
      MYSQL_PASSWORD: app
    ports:
      - "3306:3306"
    healthcheck:
      test: ["CMD", "mysqladmin", "ping", "-h", "localhost", "-proot"]
      interval: 5s
      timeout: 5s
      retries: 5

  redis:
    image: redis:7-alpine
    ports:
      - "6379:6379"
    healthcheck:
      test: ["CMD", "redis-cli", "ping"]
      interval: 5s
      timeout: 5s
      retries: 5
```

一個 `docker compose up -d` 拉起 App + MySQL + Redis 三個容器。

### 容器內 vs 本機的 Host 差異

| 環境 | DATABASE_HOST | REDIS_HOST |
|------|---------------|------------|
| 本機開發 | `localhost` | `localhost` |
| Docker 容器內 | `db` | `redis` |

> **說明**：Docker Compose 中的服務名稱（`db`、`redis`）會自動解析為容器 IP。容器內不能用 `localhost`，因為那是指容器自己。

## 動手做

### 必做

**1. 啟動完整環境**

```bash
docker compose up -d
docker compose ps    # 確認三個容器都在 running 狀態
```

預期輸出：

```
NAME                STATUS              PORTS
app                 running             0.0.0.0:3000->3000/tcp
db                  running (healthy)   0.0.0.0:3306->3306/tcp
redis               running (healthy)   0.0.0.0:6379->6379/tcp
```

**2. 閱讀設定檔**

閱讀 `database.config.ts`，回答：

- `configService.get<string>('DATABASE_HOST', 'localhost')` 的第二個參數是什麼意思？
- 為什麼 Docker 環境的 `DATABASE_HOST` 是 `db` 而不是 `localhost`？

### 延伸挑戰

1. 新增一個自訂環境變數 `APP_NAME`，在 Controller 中用 `ConfigService` 讀取並回傳：

```typescript
// .env
APP_NAME=書籍管理系統

// app.controller.ts
@Get('app-info')
getAppInfo() {
  const appName = this.configService.get<string>('APP_NAME', 'Default App');
  return { name: appName };
}
```

## 踩坑提示

| 現象 | 原因 | 解法 |
|------|------|------|
| `ConfigService.get()` 回傳 undefined | `.env` 檔放錯位置或變數名稱打錯 | 確認 `.env` 在專案根目錄，變數名稱完全一致（區分大小寫） |
| `docker compose up` 後 App 啟動失敗 | MySQL 容器尚未 ready | 等 healthcheck 通過後重試，或查看 `docker compose logs app` |
| `.env` 變數沒生效 | 忘了從 `.env.example` 複製為 `.env` | `cp .env.example .env` 後重新啟動應用 |
| 容器內連不到 DB | `DATABASE_HOST` 仍設為 `localhost` | 容器內應設為 `db`（服務名稱） |
